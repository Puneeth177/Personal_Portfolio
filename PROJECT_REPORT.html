3. Mobile Detection Hook (hooks/use-mobile.tsx)

// Custom hook for responsive behavior export function useIsMobile() { const [isMobile, setIsMobile] = useState<boolean | undefined>(undefined); useEffect(() => { const mql = window.matchMedia(`(max-width: ${MOBILE_BREAKPOINT - 1}px)`); const onChange = () => setIsMobile(window.innerWidth < MOBILE_BREAKPOINT); mql.addEventListener("change", onChange); setIsMobile(window.innerWidth < MOBILE_BREAKPOINT); return () => mql.removeEventListener("change", onChange); }, []); return !!isMobile; }

7. Possible Interview Questions & Smart Answers

Technical Questions

Q: Why did you choose React over other frameworks like Vue or Angular?
A: I chose React because of its component-based architecture, which makes the code more maintainable and reusable. React's virtual DOM provides excellent performance, and its large ecosystem offers solutions for any requirement. Additionally, React's popularity in the job market makes it a strategic choice for career growth. The unidirectional data flow also makes debugging easier compared to two-way binding in other frameworks.
Q: Explain your decision to use TypeScript instead of plain JavaScript.
A: TypeScript adds static type checking, which catches errors at compile time rather than runtime. This is crucial for maintaining code quality, especially as the project grows. It also provides better IDE support with autocomplete and refactoring tools. The type definitions serve as documentation, making the code more self-explanatory for other developers. While there's a slight learning curve, the benefits in terms of code reliability and developer experience far outweigh the costs.
Q: How did you ensure the website is responsive across different devices?
A: I implemented a mobile-first approach using Tailwind CSS's responsive utilities. I created a custom hook (useIsMobile) to detect screen sizes and conditionally render components. I tested the design across multiple breakpoints: mobile (320px+), tablet (768px+), and desktop (1024px+). I also used CSS Grid and Flexbox for flexible layouts, and implemented touch-friendly navigation for mobile users. Regular testing on actual devices ensured real-world usability.
Q: What performance optimizations did you implement?
A: I used Vite as the build tool for fast development and optimized production builds. I implemented code splitting through React Router for smaller initial bundle sizes. Images are optimized and served in modern formats. I used React Query for efficient data caching and background updates. The CSS is purged of unused styles by Tailwind, reducing bundle size. I also implemented lazy loading for non-critical components and used semantic HTML for better SEO performance.

Project Ownership Questions

Q: Walk me through your development process for this project.
A: I started by analyzing my requirements - I needed a professional portfolio that showcases my skills effectively. I researched modern design trends and decided on a glassmorphism aesthetic. I set up the project with Vite and React, then implemented the component architecture. I developed each section iteratively: hero, about, projects, and contact. I used Git for version control with meaningful commit messages. Throughout development, I tested responsiveness and performance, making adjustments as needed. Finally, I deployed to Vercel for easy hosting and continuous deployment.
Q: What challenges did you face and how did you overcome them?
A: One major challenge was ensuring consistent design across all components. I solved this by implementing a design system using Tailwind CSS and shadcn/ui components. Another challenge was optimizing performance while maintaining rich animations. I addressed this by using CSS transforms instead of layout-triggering properties and implementing efficient re-rendering strategies. Cross-browser compatibility was also tricky, especially for the glassmorphism effects, which I solved through progressive enhancement and fallback styles.
Q: How would you scale this project for a larger team or more complex requirements?
A: For team scaling, I would implement stricter TypeScript configurations, add comprehensive testing with Jest and React Testing Library, and set up CI/CD pipelines. I'd introduce state management solutions like Zustand or Redux Toolkit for complex state. For larger requirements, I'd implement a proper CMS for content management, add internationalization support, and create a component library. I'd also add monitoring tools like Sentry for error tracking and analytics for user behavior insights.

Design & UX Questions

Q: How did you approach the user experience design?
A: I focused on creating a clear information hierarchy with the most important information (skills, projects, contact) prominently displayed. I implemented smooth scrolling and intuitive navigation to guide users through the content. The glassmorphism design creates visual interest without overwhelming the content. I ensured fast loading times and responsive design for accessibility across devices. User feedback loops like hover effects and smooth transitions provide immediate feedback for interactions.
Q: Why did you choose this particular color scheme and design aesthetic?
A: I chose a dark theme with cyan and purple accents to convey a modern, tech-forward impression suitable for a software developer portfolio. The glassmorphism effect adds depth and sophistication while maintaining readability. The color scheme provides good contrast ratios for accessibility. The animated background elements create engagement without being distracting. This aesthetic aligns with current design trends while remaining timeless enough to not quickly become outdated.

Implementation Guide for Improvements

1. Adding a Blog Section

Difficulty: Medium

What to change:

  • Routing: Update `src/App.tsx` to add new routes like `/blog` and `/blog/:slug`.
  • New Pages: Create `BlogPage.tsx` and `BlogPostPage.tsx` in `src/pages/`.
  • Content: Create a `/content/posts` folder for your Markdown (`.mdx`) files.

How to implement:

  1. Install a library to parse MDX, like `mdx-bundler`.
  2. Create the new routes. The `BlogPage` will list all posts, and `BlogPostPage` will render a single post.
  3. Use Vite's `import.meta.glob` to discover and import all your posts for the listing page.

2. Adding Project Filtering

Difficulty: Easy to Medium

What to change:

  • State: In your portfolio page component, add a `useState` to store the active filter.
  • UI: Add filter buttons (e.g., `ToggleGroup` from shadcn/ui) to update the filter state.
  • Logic: Filter the projects array based on the state before rendering.

How to implement:

  1. Create an array of filter tags (e.g., ['All', 'React', 'Vite']).
  2. Render buttons for each tag. `onClick` should update the filter state.
  3. Change your render logic to: `projects.filter(p => filter === 'All' || p.tags.includes(filter)).map(...)`.

3. Adding a Functional Contact Form

Difficulty: Medium

What to change:

  • Component: Create a `ContactForm.tsx` component.
  • Libraries: Use `react-hook-form` and `zod` (already installed).
  • Service: Sign up for a client-side email service like EmailJS or Formspree.

How to implement:

  1. Build the form UI with `shadcn/ui` components.
  2. Define a `zod` schema for validation.
  3. Use the `useForm` hook with the `zodResolver`.
  4. In the `onSubmit` function, call the email service API to send the data.
  5. Use `sonner` (already installed) to show success/error toast notifications.

4. Implementing Theme Switching

Difficulty: Easy

What to change:

  • Provider: Ensure `ThemeProvider` from `next-themes` wraps your app.
  • Component: Create a `ThemeToggle.tsx` button component.

How to implement:

  1. In the toggle component, get `setTheme` and `theme` from the `useTheme` hook.
  2. Render a button that calls `setTheme(theme === 'light' ? 'dark' : 'light')` on click.
  3. Place the component in your site header.
